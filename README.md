# ubiquitous_Computing
 General information
The completion of each laboratory exercise requires the following parts:

- Lab report (submitted via Moodle).

- Lab delivery (showing the results on your laptop/PC). Not needed for Lab 4.

- Presentation (in front of the class, in group or individually, depending on the lab).

 
Lab Exercises reports (this is a part of the final grade calculation)
For each laboratory exercise, each person must submit a report as a PDF file. It consists of two parts and must be uploaded latest on the demonstration/presentation day at 23:59:

Part 1 (for Lab 1 and Lab 3): Common Part - a short description of the group work carried out (2-3 pages). Please describe the steps taken and the result. Screenshots/diagrams/figures are allowed if appropriate. Challenges/problems encountered (and their resolution) should also be included here. Do not include code parts unless they are really crucial. This part can be the same for the entire group or it can be written individually if you prefer.
Part 2 (for all Labs): Individual Part - this part will be included in the calculation of the final grade. Each laboratory exercise task description will include information on what should be included in the report. In general, you will have both theoretical and practical tasks. For theoretical tasks, you must provide the information requested in the task description (e.g. your ideas or summary of a given topic). For practical tasks (if nothing else is specified in the task description) you should write down the steps you have taken to achieve the task objective and the description (possibly with graphs/screenshots/photos) of the final result. You should also provide information about the challenges you faced and how you overcame them. 
Additional information: the number of pages indicated is only a guide. If you know that you need to write more to clearly address the task (especially if you have some figures), you can write more pages. The things you do just have to make sense and be justified.
